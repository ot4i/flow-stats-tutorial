This simple application contains a message flow which simply receives HTTP data and echoes it back to the requesting client. Once we have created an Integration Server we will deploy this example flow later in the tutorial.

Import

Prepare an Integration Server

  1. Assuming you are using a Windows platform, open an App Connect Enterprise Command Console and run the following command:
    mqsicreateworkdir C:\StatsServer
    where C:\StatsServer will the working directory for your Integration Server.
  2. Navigate to your server's working directory and locate the file 'server.conf.yaml'. Open it using your preffered text editor and set the following parameters: messageFlowMonitoring: active
    StatsSnapPublicationOn: active
    StatsSnapNodeDataLevel: advanced
    StatsSnapOutputFormat: 'json'
    StatsSnapThreadDataLevel: basic
    StatsSnapAccountingOrigin: basic
    Save and close this file.
  3. Now start the server by entering:
    IntegrationServer --work-dir C:\StatsServer --name StatsServer --admin-rest-api 7600 --http-port-number 7800 --console-log
    into an ACE Command Console.
  4. Connect to the server in the toolkit then deploy the StatsBar.bar file to the server.
  5. Right click on the server in the Toolkit and launch the WebUI.

Deploy

Follow these steps to complete the tutorial

Follow the following steps to complete the tutorial.

  1. Navigate to a Command Prompt and run this command a handful of times:
    curl -X POST http://localhost:7800/request -d HelloWorld
  2. Assuming you did not get a 404 from the server, navigate to the WebUI for the server.
  3. Navigate to the 'Statistics' tab on the navigation bar, then in the drop-down menu on the left select 'Message flow statstics' and wait a moment.
  4. After a brief pause, you should see some records labelled 'Request' and 'internalReply' with various statistics about them. Notice that the values in the 'Total input messages' fields corresponds directly to the amount of curl requests you made to the server.
  5. To get a more detailed view of how data is passing through your flows, change the view to 'Flow node statistics' in the 'Showing' drop-down.
  6. Each record in the previous screen corrsponded to an entire Flow, while in this view each record represents an individual Node in a Flow. On the right hand side you can see which Flow each Node belongs to.
  7. When you're ready to conclude this tutorial, stop the Integration Server in the Command Console and close your browser.
